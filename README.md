# Тренажёр для подготовки к экзаменам (Edoox)

## Требования для запуска
Для работы приложения необходим ключ API от Google Gemini.
Добавьте ваш ключ в файл `local.properties` в корне проекта:
```properties
GEMINI_API_KEY=your_api_key_here
```

## Решения и компромиссы

### UI Фреймворк
Использован Jetpack Compose, так как это современный стандарт Android-разработки. Он отлично подходит для динамических форм (вопросы разных типов), так как позволяет избегать сложных XML-раскладок и RecyclerView.

### Схема JSON для заданий
При генерации мы требуем от LLM строгий формат с использованием `responseSchema` (feature Gemini API).
Схема описывает типы вопросов (`CLOSED`, `OPEN`, `MATCHING`), а также специфичные для них поля (`options`, `pairs`, `correctAnswer`).

### Проверка трёх типов вопросов
- **Закрытые (CLOSED)**: Проверяются локально на клиенте простым сравнением строки ответа со строкой `correctAnswer`.
- **На соответствие (MATCHING)**: Проверяются локально. Словарь (Map) пользовательских ответов сравнивается со словарем эталонных пар (`concept` -> `definition`).
- **Открытые (OPEN)**: Для проверки отправляется второй запрос к LLM (модель проверяет ответ пользователя на основе эталонного ответа и текста вопроса). 

### Обработка ошибок
- Ошибки при сетевом запросе или парсинге JSON отлавливаются в `catch` блоке внутри корутины.
- Пользователь видит экран "Ошибка" с описанием проблемы и кнопкой возврата на главный экран, что предотвращает краш приложения.
- Если сгенерировался неполный список вопросов (что редко бывает благодаря `responseSchema`), мы показываем то количество, которое удалось распарсить (но в промпте мы жестко требуем 5).

### Архитектура (MVVM)
- **Data Layer**: `TestRepository` инкапсулирует логику работы с Gemini SDK.
- **UI Layer**: `SharedTestViewModel` управляет состоянием экранов (генерация, процесс теста, проверка результатов) через `StateFlow`.
- **DI**: В рамках тестового задания использован простой ручной DI (передача репозитория в конструктор ViewModel по умолчанию). Это компромисс для экономии времени на настройку Dagger/Hilt.
